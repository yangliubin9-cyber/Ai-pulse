[技术栈]
═══════════════════════════════════════════════════════════════

- 前端：React 18 + TypeScript（strict）+ Vite 5 + Tailwind CSS + shadcn/ui 复制件 + TanStack Query v5 + React Router v6（data router）
- 后端：Python 3.12 + FastAPI + SQLAlchemy 2（async）+ Alembic + Pydantic v2 / pydantic-settings
- 采集：feedparser（RSS）+ httpx（HN / arXiv 官方 API）；定时用 FastAPI 启动期后台循环 + 手动触发端点（不引入 Celery/MQ）
- 部署方式：**必须支持 docker-compose / GitLab / Helm 三种**（铁律三，不可选）
- 包管理：前端 npm（package-lock 提交）、后端 uv（uv.lock；本机无 uv 时 venv+pip 兜底）
- ORM/测试：SQLAlchemy 2 async / 后端 pytest + 前端 Vitest + RTL

[数据保护 — PII Field Catalog]（铁律八）
═══════════════════════════════════════════════════════════════

个人私有部署，仅登录账号涉及 email。资讯内容为公开来源转载（含原文链接与署名来源）。

| 字段                | 表 / 集合       | 等级 | 加密方式 | 审计 | 留存期 |
|---------------------|-----------------|------|----------|------|--------|
| users.email         | postgres.users  | L1   | 无（项目级豁免：单用户私有部署，账号即所有者本人，库不出本机） | 登录失败锁定计数 | 随账号生命周期 |
| users.password_hash | postgres.users  | 凭证（非 PII） | scrypt 单向哈希 | - | 随账号生命周期 |
| user_item_states.*（user_id/item_id/saved/read） | postgres.user_item_states | L1（行为数据，非身份信息） | 无（项目级豁免同上：单用户私有部署、账号即所有者本人、库不出本机） | - | 随账号生命周期 |

- **跨境部署**：☑ 禁止（默认）
- **合规对接**：无（个人使用）；资讯转载保留来源出处与原文链接，仅展示来源提供的标题/摘要，不全文转载正文
- **DPIA**：☐ 不需要

═══════════════════════════════════════════════════════════════
[项目专属规则 / 覆盖项]
═══════════════════════════════════════════════════════════════

- **登录鉴权**：Session Cookie（HttpOnly+SameSite=Lax，session 存 Redis，TTL 7 天），密码 scrypt 哈希（标准库）。默认账号待定（首次启动 seed，启动日志告警改密；设置页提供改密）。
- **v1 不接入 LLM**：摘要直接用来源原文（RSS summary / HN 标题 / arXiv abstract）；分类（模型/产品/行业/论文/技巧）用关键词规则打标。"精选度评分""推荐理由"等需要 LLM 的功能列为 v2，先不做。
- **数据合规红线**：只从公开 API / RSS 抓取；尊重 robots 与各源 ToS；HN/arXiv 用官方 API；只存来源标题/摘要/链接/发布时间/来源名，不抓取并转载第三方编辑的精选内容（不抄 AIHOT 等站点数据）。配图走 img-proxy 代理或不存。
- **采集去重**：以来源 URL 规范化后做唯一键，避免重复入库。
